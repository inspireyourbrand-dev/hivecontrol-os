Spawn Full Team Playbook

Activate the complete HiveControl agent swarm with this command flow:

Prerequisites

  • HiveControl OS environment initialized
  • All 8 permanent agents configured and connected
  • WebSocket server running for agent communication

Activation Command

hivecontrol spawn-team --full --cascade

Command Flow

  1. Initialize Orion - Master orchestrator boots and establishes coordination hub
  2. Bootstrap Specialists - All 7 specialist agents initialize in parallel:
    • Atlas (Infrastructure & Ops)
    • Forge (Code & Build)
    • Patch (Bug Fixing)
    • Quill (Content & Docs)
    • Cipher (Security & Compliance)
    • Pixel (Design & UI/UX)
    • Spark (Research & Analysis)
  3. Verify Connectivity - All agents confirm WebSocket connection and readiness
  4. Load Capability Maps - Each agent loads its capability definitions and trigger conditions
  5. Establish Work Queue - Orion sets up distributed task queue for incoming work
  6. Enter Ready State - Swarm enters standby, waiting for work triggers

Status Verification

Monitor activation progress:

hivecontrol team status --watch

Expected Output

Once complete, all 8 agents should show ACTIVE status with zero drift. The swarm is then ready to process work continuously.

Note: The team playbook is automatic. Additional dynamic agents will spawn on-demand based on workflow complexity.
